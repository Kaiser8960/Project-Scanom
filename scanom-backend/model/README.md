# Model Files — Scanom Backend

Place the following files in this directory before starting the backend:

## Required Files

| File | Source | Size (approx) |
|---|---|---|
| `efficientnetv2b0.tflite` | Run `ml/export_tflite.py` in WSL2, then copy here | ~25 MB |
| `class_names.json` | Generated by `ml/train.py`, found in `model_output/` | < 1 KB |
| `best_model.keras` *(optional fallback)* | Found in `model_output/` | ~50 MB |

## How to Copy from WSL2

From WSL2 Ubuntu terminal:
```bash
# Export TFLite
source ~/scanom-ml-env/bin/activate
python3 "/mnt/c/Users/Jim Dejito/OneDrive/Desktop/Jim Codes/Thesis/Scanom/ml/export_tflite.py"

# The TFLite file will be at:
# /mnt/c/.../Scanom/model_output/efficientnetv2b0.tflite
# Copy it here (the backend's model/ folder)
```

## class_names.json Format

```json
[
  "banana_cordana",
  "banana_healthy",
  "banana_panama_wilt",
  "banana_sigatoka",
  "tomato_bacterial_spot",
  "tomato_early_blight",
  "tomato_healthy",
  "tomato_late_blight",
  "tomato_leaf_mold",
  "tomato_powdery_mildew"
]
```

> Order MUST match the order of classes during training (as output by split_dataset.py).
> Check `model_output/class_names.json` to confirm.
